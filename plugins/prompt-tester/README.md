# prompt-tester

**Run your prompts. Check they work. Catch regressions.**

Executes the `tests.json` generated by prompt-crafter. Each test case is run through the prompt, and output is checked against `expected_contains` assertions. Reports pass/fail per test case with detailed failure reasons.

## Why

Flux generates `tests.json` with every prompt, but nothing EXECUTES those tests. This plugin closes the loop — you can now verify a prompt actually works before deploying it, and catch regressions after refinements.

## Usage

```
/test-prompt                          # pick from saved prompts
/test-prompt stocks-analysis          # test a specific prompt
/test-prompt stocks-analysis --ci     # CI mode (JSON output, exit code)
```

## What Happens

```
PROMPT TEST RESULTS: stocks-analysis
Target model: claude-opus-4-6

  PASS  single-stock-output-structure (6/6 assertions)
  PASS  data-labels-present (2/2 assertions)
  FAIL  invalid-ticker-edge-case (0/1 assertions)
        ✗ expected "not recognized" — not found

RESULT: 2/3 passed (67%)
```

## Output

Saves `test-results.json` to the prompt folder with timestamps, pass rates, and per-assertion detail.

## Components

| Type | Name | Model |
|------|------|-------|
| Skill | test-runner | (main agent) |
| Agent | executor | Sonnet — runs individual test cases |
